# [5-QA] API Design Review

> **Stage**: Quality Assurance | **Category**: API Quality | **Frequency**: API changes

Audit API design for consistency and best practices. Track in `TODO.md`.

## Review Criteria

1. **RESTful Compliance**: Resource naming, HTTP methods, status codes
2. **Consistency**: Naming conventions, response formats, error handling
3. **Versioning Strategy**: Breaking change management
4. **Documentation**: OpenAPI/Swagger completeness
5. **Security**: Auth mechanisms, rate limiting, input validation

## Output

- `docs/api-review.md`
- Updated OpenAPI spec (if applicable)
- Migration guide for breaking changes
